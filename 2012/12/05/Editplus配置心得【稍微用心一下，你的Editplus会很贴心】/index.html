Editplus配置心得【稍微用心一下，你的Editplus会很贴心】

发表于 2012-12-05   |  

Editplus是一款轻量的代码编辑软件，小巧快速，真的是爱不释手啊~ 不过如果你能够根据你的工作改造一下，那真的就是完美了~

首先要说的是Editplus具有很强的扩展性，有时候并不比那些臃肿的IDE弱多少~ 不信么？来看看吧~

1.让Editplus支持HTML5、CSS3

HTML5和CSS3盛行的时代，你的Editplus怎么能不支持HTML5和CSS3呢？作为一名前端er这个是很重要的~

让Editplus支持HTML5和CSS3无非就是支持一些HTML5和CSS3的代码高亮，以及自动填充的功能。首先，打开你的Editplus，依次点击工具-参数设置-文件-设置&语法，选择HTML文件类型，然后会看到地下有语法文件和自动完成。语法文件是以.stx结尾的，看到这里你可能已经想到了点什么。是的，在网上下载一个html5.stx然后在Editplus里设置好HTML文件的语法文件关联，然后就OK了。这样在你写出HTML5特有的一些标签，比如<seciton><header><footer>的时候，代码也会高亮了。同理，CSS3也是这样支持代码高亮的。然后呢就是自动完成了。这个真的是一个很好很好的东西~

2.让你的Editplus更加人性化

很多编辑器以及IDE（集成开发环境）都会有自动完成的功能，比如你要输入background，很可能你输入bak然后空格就会自动给你把剩下的写上了，恩，就是这个样子的，很不错是吧。同理，Editplus也是支持这个功能的，现在按照第一步的那样设置，然后会找到自动完成的开关选项以及关联文件，是以acp结尾的文件。哈哈，是不是同理了呢？直接下载别人编辑好的acp然后自己用就OK了。以我的acp文件为例，给大家讲解一下都会有什么功能吧~

可以看到有很多#T=*，这个就是缩写的意思，比如我会把HTML5的一些常用的格式会作为一个块，然后每次输入HTML5空格，一大片代码就自动出现了，恩就是这么方便。爽吧，需要用到jQuery的时候直接JQUERY空格~引入CSS文件的时候也只需要LINKSTYLE空格~ 确实是很不错的东西。

3.配置用户工具

其实Editplus远不止有这些功能，还有一个比较实用的就是配置用户工具，这个功能也很实用的说~依次点击工具-配置用户工具，然后添加工具

SVN Commit

C:\Program Files\TortoiseSVN\bin\TortoiseProc.exe

/command:commit /path:”$(FilePath)” /closeonend:0

$(FileDir)

上面那个就是四行需要自己输入的，这个工具就是配置了SVN，编辑好文件后只需要Ctrl+1，然后会自动调出SVN，你只需要点一下确定，然后就开始同步啦~

Current Location

%systemroot%\explorer.exe /e,/root,\local disk,

$(FileDir)

这个是打开文件所在文件夹的命令，直接就可以Ctrl+2打开文件所在目录了，是不是很方便呢。

4.一劳永逸，保存你的成果。

另外还可以配置Editplus的背景颜色为淡淡的黄色（245,233,211），字体配置为Consolas，Regular字号为11。这样的背景颜色刚好各个颜色的代码都可以看清楚，而且眼睛看着会舒服一点。总比全部是白色的还看多了，另外这个Consolas字体也被公认为写代码最漂亮的字体~
在配置完成后，为了避免重装系统导致的配置信息丢失，可以在Editplus安装目录里把四个ini配置文件备份，以及你的acp自动完成文件和.stx语法文件备份~恩，就是这么方便~！
