Apahce同IIS并存以及Apache虚拟机的配置

发表于 2013-10-23   |  

废话不多说，直接上代码。最近比较忙，都没时间写博客了。

1.首先在Apache的目录里找到\conf\httpd.conf修改配置文件，搜索Proxy，将下面的几行配置信息前的#去掉，使配置生效。

LoadModule proxy_module modules/mod_proxy.so
LoadModule proxy_connect_module modules/mod_proxy_connect.so
LoadModule proxy_http_module modules/mod_proxy_http.so
LoadModule proxy_ftp_module modules/mod_proxy_ftp.so

这几行是加载必要的运行模块。
2.在httpd.conf最后边添加下面这行代码。

Include \conf\extra\httpd-vhosts.conf3.在\conf\extra\httpd-vhosts.conf修改成以下的样例
<Directory ../vhosts>
AllowOverride All
Order allow,deny
Allow from all
</Directory>
NameVirtualHost 
<VirtualHost >
DocumentRoot ../htdocs
ServerName default:80
ErrorLog logs/default-error_log
</VirtualHost>
<VirtualHost :80>
ServerAdmin webmaster@yourdomain.com
DocumentRoot ../htdocs/yourdirctory
ServerName www.yourdormain.com
ErrorLog “logs/demo.neoease.com-error.log”
CustomLog “logs/demo.neoease.com-access.log” common
</VirtualHost>
<VirtualHost :80>
ServerName www.youriiswebsite.com
ProxyPass / http://127.0.0.1:8001/
ProxyPassReverse / http://127.0.0.1:8001/
</VirtualHost>

根据以上的案例进行修改。具体来说使用VirtualHost可以自定义某域名的初始目录，比如www.yourdomain.com可以映射到../htdocs/yourdirctory，而同时也可以使用VirtualHost将来自www.youriiswebsite.com的请求全部转发到http://127.0.0.1:8001这个端口上。这样就可以做到IIS与Apache的共存。其实就是IIS来用其他端口，使用Apache监听所有的80端口，并对来自特殊域名的请求予以转发到特定的IIS的端口，也就是Apache做前端代理。这样就能达到一个服务器放多个网站以及一个服务器同时安装IIS和Apache并且不相互冲突的目的了。写的比较粗糙，有疑问留言，更多的是想让自己更加清楚。
