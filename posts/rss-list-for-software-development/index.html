软件开发相关 RSS 源推荐

2020年3月8日

RSS 作为一种开放的内容聚合平台，发展到今天在微信公众号、今日头条的围剿之下已然变成了一种小众的方式。但这种小众的方式却依然存留着自己独特的魅力，在信息洪流的时代下像是一股清流，静静地呆在那等你想起来的时候去看几篇，没有广告，没有所谓的智能推荐。今天就来给大家推荐一些我自己订阅并真正阅读的一些关于软件开发的 RSS 源。

RSS 客户端

我在 Mac 和 IOS 上使用的是 Reeder 作为阅读客户端。内容聚合用的是 feedly 的账户，免费版感觉已经够用了。曾经有过想要把 RSS 推送到 Kindle 的念头后来转念一想 Kindle 还是更适合看书一些，RSS文章更适合手机这种碎片化的场景，无聊了随时掏出手机就能看，也是一种代替刷朋友圈、微博的好办法。

中文源推荐

阮一峰的网络日志

网上有很多人对阮一峰的评价褒贬不一，诚然可能部分文章有一些瑕疵但大部分文章我认为写的非常好，通俗易懂是最大的特点。最喜欢的栏目是《科技爱好者周刊》，每周五发稿，这个栏目有点像是一道甜点，不那么枯燥更像是一道甜点，每次都不知不觉的能看完。

网站：http://ruanyifeng.com/blog

RSS：http://www.ruanyifeng.com/feed.html

当然我在扯淡 - 王垠

当然，又是一个褒贬不一的博主。不过那又有什么所谓呢，自己喜欢就好。王垠的很多计算机科学相关文章我非常喜欢，甚至在看到文章中有付款二维码时也会打开支付宝进行付款之后再阅读观看。除此之外一些生活观、价值观之类的网站我也比较赞同，赞同区分开赞助和收费以及自己制作的内容有权收费这样的观点。

网站：http://www.yinwang.org/

RSS: 无官方 RSS 源

张鑫旭-鑫空间-鑫生活

最早接触张鑫旭的文章是自己还在学 CSS 的时候，正是他的文章让我搞懂了 CSS Flex 的布局。不得不说我觉得张鑫旭是个很聪明踏实的人，能够日复一日年复一年的坚持学习和写作（可能还有钓鱼？），正如他所说的一样：

技术比人情更可靠，你所学习的技术知识，积累的那些细节和经验，百分之一百日后可以给你带来令你欣喜的价值，而且这个价值连绵不绝，持续不断，越筑越高。因为技术不像人，技术不会欺骗你，而且越基础的技术越忠诚。

网站：https://www.zhangxinxu.com/

RSS: https://www.zhangxinxu.com/wordpress/feed/

ThoughtWorks 洞见

洞见并不是一个私人博客，而是 ThoguhtWorks 内部的投稿平台。有幸参加过洞见主编组织的写作课，也知道了一些洞见平台投稿的规则等。总体来说洞见的文章质量都相对较高，融合了各种一线开发的经验和思想。内容方面很多文章都写的非常专业，以至于我最开始看的时候觉得部分文章很难看懂，但后来我才发现一篇介绍某个具体的技术、工具这样的文章虽然看起来比较简单易懂但更多的是没有“思想”融入的文章。往往更高质量的文章是夹杂着自己的思考，所谓有想法、有思考、说服自己。

网站：https://insights.thoughtworks.cn/

RSS: https://insights.thoughtworks.cn/feed/

哈哈，这是私货！！！ 不过连自己的博客都不舍得推荐那怎么鼓励自己坚持写下去呢？

英文源推荐

martinfowler.com

Martin Fowler 的博客建立的时候我才两岁，这个博客为什么这么牛逼呢？因为他是被称作“书写开发哲学”的男人啊。你可能没看过他的博客，但是你可能听过或者看过他写的这些书：《重构》、《企业应用架构模式》、《领域特定语言》、《UML精髓》。从敏捷到重构再到微服务、微前端，Martin 的博客简直就像一个软件开发的宝库，值得自己每周花几个小时去细细品味。当然除此之外他的演讲也非常精彩，建议上 youtube 上观看。

网站：https://martinfowler.com/

RSS: https://martinfowler.com/feed.atom

Overracted

Redux 的作者 Dan Abramov 自己的个人博客，虽然建立没有多久但是里面讲述了不少自己对于 React/Redux 的理解，另外也时不时会发发牢骚或是回顾一下自己的十年，讲述自己开始学习编程以及如何追到女票的故事。

网站：https://overreacted.io/

RSS: https://overreacted.io/rss.xml

2ality - Javascript and more

一个写 Javascript 的男人，Axel 总是能深入到 JS/TS 的规范细节中，通过简单通俗的例子讲解某个 API 的用法，我为什么喜欢看他的博客而不是官方手册呢？因为有趣啊！时不时带一些私货告诉你原来这个方法还能这么玩！！除此之外他也写了好几本关于 JS 以及 ReasonML 的书。对 JS 以及 ECMA 规范感兴趣想要跟上 ES20xx 的潮流的话看他的博客准没错。

网站：https://2ality.com/

RSS: https://feeds.feedburner.com/2ality

web.dev

web.dev 是一个专门讲述 Web 开发的专业博客，涉及性能优化、Accessibility、PWA 等多方面的博客。除了博客之外这个网站也有很多 Web 开发相关的资源，前端开发可以关注一波。

网站：https://web.dev/blog/

RSS: https://web.dev/feed.xml

Sacha Chua

一个 Emacs 深度用户的博客，会 Weekly 地更新 Emacs New 以及晒出自己的画的一些草图，从博主的博客可以看出来她是个非常 Geek 并且自律的人，Emacs 爱好者可以关注一波。

网站：https://sachachua.com/blog/

RSS: https://sachachua.com/feed/

官方没有 RSS 源怎么办？

很多博客或是像简书这样的网站没有 RSS 源但是你又想订阅，这种情况下 Feedly 中直接输入网址，有时候会自动帮你找出 RSS 源。如果 Feedly 也没有办法找出源那就需要用到 rsshub 这样的服务通过类似于爬虫的方式解决源的问题。

写在最后

用 RSS 替代刷朋友圈微博实际上是个很好的主意，如果你经常读一个人的博客你会发现当你真的见到他以后你会觉得异常的熟悉，即便这是你们的第一次见面。所以我认为博客同样也是社交的一种方式，甚至是另外一个更深层次的社交方式让你真正更深入的了解一个人的想法。而且这些想法一旦发布出来之后就几乎没有什么成本，永远的留在互联网上，何乐而不为呢？
